# API Documentation - Whoosh Ticket App

This documentation is updated based on the current implementation of the backend.

## Base URL: `/api`

### 1. Authentication
*   **POST** `/auth/register` - Register a new user.
    *   Body: `{ full_name, email, phone, password }`
*   **POST** `/auth/login` - Login and get JWT token.
    *   Body: `{ email, password }`
*   **GET** `/auth/me` - Get current user info (Requires Token).

### 2. User & Profile (Requires Token)
*   **GET** `/users/profile` - Get user profile details.
*   **PUT** `/users/profile` - Update user profile.
    *   Body: `{ full_name?, email?, phone? }`
*   **DELETE** `/users/profile` - Delete user profile.

### 3. Saved Passengers (Requires Token)
*   **GET** `/saved-passengers` - Get list of saved passengers for the user.
*   **POST** `/saved-passengers` - Save a new passenger to user's list.
    *   Body: `{ full_name, id_number }`
*   **DELETE** `/saved-passengers/:id` - Delete a saved passenger.

### 4. Stations
*   **GET** `/stations` - Get all available stations.
*   **GET** `/stations/:id` - Get details of a specific station.
*   **POST** `/stations` - Create a new station (Requires Admin).
    *   Body: `{ station_name, location }`
*   **PUT** `/stations/:id` - Update station (Requires Admin).
*   **DELETE** `/stations/:id` - Delete station (Requires Admin).

### 5. Trains
*   **GET** `/trains` - List all trains.
*   **GET** `/trains/:id` - Get train details.
*   **POST** `/trains` - Create a new train (Requires Admin).
    *   Body: `{ train_name, train_code, total_seats }`
*   **PUT** `/trains/:id` - Update train (Requires Admin).
*   **DELETE** `/trains/:id` - Delete train (Requires Admin).

### 6. Seats
*   **GET** `/seats/train/:trainId` - Get seats by train.
*   **GET** `/seats/train/:trainId/class/:class` - Get seats by train and class.
*   **POST** `/seats/train/:trainId` - Create a new seat (Requires Admin).
    *   Body: `{ seat_number, class }`
*   **DELETE** `/seats/:id` - Delete a seat (Requires Admin).

### 7. Schedules
*   **GET** `/schedules` - List all schedules.
    *   Query Params: `departure`, `arrival`, `date`
*   **GET** `/schedules/:id` - Get specific schedule details.
*   **POST** `/schedules` - Create a new schedule (Requires Admin).
    *   Body: `{ train_id, departure_station, arrival_station, departure_time, arrival_time, price }`

### 8. Bookings (Requires Token)
*   **POST** `/bookings` - Create a new booking.
    *   Body: `{ schedule_id, passengers: [{ full_name, id_number, seat_id? }] }`
*   **GET** `/bookings/my` - Get user's bookings.
    *   Query Param: `type` (`unpaid`, `paid`, `history`)
*   **GET** `/bookings/:id` - Get booking details.
*   **POST** `/bookings/:id/cancel` - Cancel a booking.

### 9. Payments (Requires Token)
*   **POST** `/payments/booking/:bookingId` - Process payment for a booking.
    *   Body: `{ payment_method, amount }`
*   **PUT** `/payments/:paymentId/status` - Update payment status.
    *   Body: `{ payment_status }`
